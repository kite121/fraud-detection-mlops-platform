Date: 27.04.26
Tester: Amir Valeev
Branch: fixed_final_branch
Environment: Local (Windows 11)

Goal:
- Verify that Prometheus exposes and updates core platform metrics after API activity, training, and inference.

Preconditions:
- The platform is running in Docker.
- Prometheus is available at http://localhost:9090.
- worker-service metrics endpoint is enabled and scraped by Prometheus.
- At least one new training run and one new predict request were executed after the metrics fix.

Steps performed:
1. Rebuilt and restarted ingestion-service, inference-service, worker-service, and prometheus after the metrics fix.
2. Executed a new training run and a new predict request to generate fresh metric values.
3. Opened Prometheus and ran queries for HTTP, training, model-registration, inference, and active-model metrics.

Expected result:
- HTTP request metrics are present.
- Training job and training duration metrics are present and non-zero after a completed training run.
- Model registration metrics are present.
- Inference request and duration metrics are present and non-zero after a predict call.
- active_model_version_info shows only the currently active model version.

Actual result:
- http_requests_total returned non-empty results for ingestion-service and inference-service endpoints.
- training_jobs_total showed both queued=1 and completed=1.
- training_duration_seconds_count returned 1 and training_duration_seconds_sum returned 32.43245003500033.
- models_registered_total returned status="validated" with value 1.
- inference_requests_total returned 1.
- inference_duration_seconds_count returned 1 and inference_duration_seconds_sum returned 0.03887655300059123.
- active_model_version_info returned only model_version="v003" with value 1.
