В модуле реализован evaluation layer для Sprint 2, который оценивает качество обученной fraud-модели на validation data и возвращает метрики в структурированном виде. В качестве основного дизайнерского решения метрики вынесены в отдельный модуль, а не встроены внутрь training.py: это позволяет независимо сравнивать версии моделей, переиспользовать одну и ту же оценку для разных baseline-алгоритмов и хранить результат в model registry как отдельную сущность.

Функция `evaluate_model()` принимает либо готовую модель, либо `TrainedModelResult`, получает probabilities для positive class и считает набор метрик, подходящих именно для fraud detection: `ROC-AUC`, `Precision`, `Recall`, `F1-score`, `PR-AUC`, а также summary по confusion matrix. В коде основной metric по умолчанию зафиксирована как `pr_auc`, потому что для сильно несбалансированных fraud datasets она обычно лучше отражает качество работы модели на редком положительном классе, чем обычная accuracy. В результате evaluation layer возвращает компактный объект с именем модели, выбранной primary metric, значением этой метрики, validation statistics и полным набором рассчитанных показателей, чтобы их можно было дальше сохранять, сравнивать и использовать при выборе лучшей модели.
